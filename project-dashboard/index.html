目前不建議再動正式 code。下一步是使用者在實機上驗收 H-wide Explorer。

建議日期：2026-07-07 或使用者實機時

人工驗收清單

  • 打開 H-wide Explorer，確認可看到 H 根目錄主要資料夾。
  • 點進 3 到 5 個一般資料夾，並返回上一層。
  • 點 Windows 開啟資料夾。
  • 開照片、PDF、影片、Word、Excel、PowerPoint。
  • 在安全測試資料夾測新增資料夾、改名、複製貼上、剪下移動。
  • 測移到 H 安全回收區，不做永久刪除。
  • 確認系統保護區不可操作。
  • 確認 F 沒被碰，且 F/H 原始資料同步沒有執行。

若人工驗收通過

  • 進入 H-only 正式入口整合階段。
  • 目標：H 啟動後可直接進 H-wide Explorer。
  • 仍只做 H，不碰 F，不做 F/H 同步。
  • 不合併 main、不升正式版，除非使用者另外批准。
  • 保留 rollback、系統保護區、H 安全回收區。
  • 保留既有 /data、/file、/open-file、/open 路由。

安全規則與每次更新格式

這些規則是讓公開看板能長期用下去的安全邊界。

固定規則

禁止放進 public repo

  • F/H 原始資料、照片、影片、Office、PDF。
  • 真實 driveId、磁碟代號、完整本機路徑。
  • SQLite、logs、runtime、cache、temp、本機測試輸出。
  • deploy key、token、密碼、secrets、未遮蔽截圖。

每次更新 Dashboard

  • 需求、資訊、風險或驗收狀態有變更時自動同步。
  • 分清楚最新消息與過去紀錄。
  • 回報本次進 GitHub、明確排除與分類不確定項目。
  • 分類不確定時停止詢問，不自行公開。
每次更新固定欄位：
- 日期 / 分支 / commit
- 最新消息
- F 狀態 / H 狀態
- 是否有 F/H 原始資料同步：通常應為「沒有」
- Dashboard 是否已同步
- 本次進 GitHub
- 明確排除
